• 操作系统: Windows 7/8/10/11 (64位)
• 内存: 至少 2GB RAM
• 硬盘: 至少 100MB 可用空间
• 网络: 需要互联网连接以监控远程服务器

【使用方法】
1. 首次运行
   • 双击运行 "ServerMonitor.exe"
   • 程序会自动创建数据库和密钥文件

2. 添加服务器
   • 切换到"服务器管理"标签页
   • 填写服务器名称、地址和访问密钥
   • 点击"测试连接"验证配置
   • 点击"添加服务器"保存

3. 配置监控参数
   • 设置CPU、内存、负载阈值
   • 设置检测间隔时间
   • 配置智能告警参数
   • 点击"保存配置"

4. 开始监控
   • 切换到"服务器监控"标签页
   • 点击"开始监控"按钮
   • 实时查看服务器状态
   • 查看监控日志

【配置说明】
⚙️ 监控阈值
   • CPU阈值: 建议80-90%
   • 内存阈值: 建议85-90%
   • 负载阈值: 建议80-90%

⚙️ 检测间隔
   • 最小5秒
   • 建议15-30秒
   • 根据服务器数量调整

⚙️ 智能告警
   • 启用后进行连续验证
   • 验证次数: 建议3-10次
   • 验证间隔: 建议1-2秒
   • 时间窗口: 建议600-1800秒

【数据文件】
📁 monitor.key
   • 加密密钥文件
   • 首次运行自动生成
   • 用于加密数据库中的敏感信息

📁 server_monitor.db
   • SQLite数据库文件
   • 首次运行自动创建
   • 存储服务器配置和设置

【注意事项】
⚠️ 重要提示
   • 请妥善保管 monitor.key 文件
   • 不要随意修改数据库文件
   • 定期备份配置数据
   • 确保网络连接稳定

⚠️ 安全建议
   • 使用强密码保护服务器
   • 定期更换访问密钥
   • 不要在公共网络使用
   • 及时更新软件版本

【常见问题】
Q: 程序无法启动？
A: 检查系统是否为64位，尝试以管理员权限运行

Q: 无法连接服务器？
A: 检查服务器地址、密钥是否正确，网络是否通畅

Q: 告警通知不显示？
A: 检查系统通知权限，确保未被禁用

Q: 数据库文件损坏？
A: 删除 server_monitor.db，程序会自动重建

Q: 密钥文件丢失？
A: 删除数据库文件，重新配置服务器
